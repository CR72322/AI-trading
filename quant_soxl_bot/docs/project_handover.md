# 项目概况
这是一个基于 Alpaca API 的美股（特别是 SOXL）量化交易回测系统。

# 核心目录结构
- `src/strategies/volatility_trend.py`: 核心交易策略逻辑（当前为波动率趋势策略）。
- `src/utils/alpaca_loader.py`: 负责从 Alpaca 拉取历史 K 线数据。
- `src/backtest/run_optimization.py`: 负责执行网格搜索，寻找最优的策略参数。

# 历史上下文与当前痛点
在最近的测试中，策略在 SOXL 暴涨的单边牛市中依然全线亏损。
诊断结论：原有的 EOD（下午 15:55 日内强制平仓）逻辑，导致我们错过了大量的隔夜跳空高开利润，并在日内承担了无意义的震荡磨损。

# 下一步目标 (Experiment 006)
1. 切换到更高周期的数据（从 5 分钟 K 线改为 15 分钟 K 线）以过滤噪音。
2. 彻底移除 EOD 强制清仓逻辑，允许持仓过夜。
3. 调整配套的止损和指标参数。
